# Risk project — портфельная оптимизация и границы эффективности (MOEX)

Учебный проект по курсу риск-менеджмента: оценка ожиданий и ковариаций доходностей российских акций, построение **границ эффективных портфелей** при разных ограничениях, динамика границ во времени, бета-факторные модели, бонусные блоки (Black, Монте-Карло, max-risk frontier, Implementation Shortfall).

---

## Важно: структура репозитория

| Что | Где |
|-----|-----|
| **Основной код и анализ** | [`project.ipynb`](project.ipynb) — весь проект собран в одном ноутбуке |
| **Вспомогательные функции** | [`risk_utils.py`](risk_utils.py) — оценки μ, Σ, окна, границы, оптимизация |
| **Графики** | папка [`plots/`](plots/) — все `plt.savefig(...)` направлены сюда |
| **Исходные данные** | папка [`data/`](data/) — размещайте CSV с ценами (см. ниже) |
| **Промежуточные результаты** | `results.xlsx` в корне (генерируется ноутбуком при прогоне соответствующих ячеек) |

> Пожалуйста, используйте **pandas версии 3.0** и выше. Зависимости зафиксированы в [`requirements.txt`](requirements.txt).

---

## Быстрый старт

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
jupyter notebook project.ipynb
```

Первую ячейку с импортами выполните до остальных. Дальше следуйте порядку разделов в ноутбуке.

---

## Данные (`data/`)

1. Скопируйте в **`data/`** широкую таблицу дневных цен закрытия (как в задании), например:
   - `data/moex_close_prices_wide_2015_2025.csv`
2. В ноутбуке пути к CSV должны указывать на **`data/...`** (при необходимости замените строку `pd.read_csv('moex_...')` на `pd.read_csv('data/moex_...')`).
3. Файл **`results.xlsx`** создаётся при выполнении блоков с экспортом μ/Σ — его можно пересобрать, запустив соответствующие ячейки.

---

## Что внутри `project.ipynb` (логика разделов)

- **Сбор данных** — API MOEX ISS, формирование рядов цен и доходностей.
- **П. 2–4** — скользящие и расширяющиеся окна, EW-взвешивание; экспорт в `results.xlsx`.
- **П. 5–8** — границы эффективных портфелей: без шортов, с шортами, с ограничениями на доли и т.д.
- **П. 9–12** — динамика границы, индексы для β, сравнение сценариев (графики в `plots/`).
- **П. 13–20** — ковариации через β (historical / adjusted), сравнение границ.
- **Бонусы:** теорема двух фондов **Black**, **«наиболее рискованные»** портфели (gross leverage, pie chart), **Implementation Shortfall** в целевой функции оптимизатора, **Монте-Карло** границы.

Подробный текст для презентации по бонусам: [`presentation_bonus_after_22.md`](presentation_bonus_after_22.md).

## Графики (`plots/`)

Все иллюстрации сохраняются в **`plots/`** (создайте каталог при первом запуске или выполните ячейку с `os.makedirs("plots", exist_ok=True)`). Типовые имена:

| Файл | Содержание (по смыслу ноутбука) |
|------|--------------------------------|
| `plot_1.png` … `plot_10.png` | Этапы анализа границ и ограничений |
| `frontier_rolling.png`, `frontier_expanding.png`, `frontier_comparison.png`, `frontier_ew_comparison.png` | Динамика границ (п. 9–10) |
| `index_comparison.png`, `beta_stability.png` | Индексы и β (п. 11–12) |
| `efficient_frontier_adj_betas_2019.png` | Граница с adjusted betas |
| `max_risk_frontier_shorts_2019.png`, `max_risk_pie_2019.png` | Max-risk frontier и структура портфеля |
| `implementation_shortfall_optimizer.png` | Оптимизация с штрафом IS |
| `monte_carlo_*.png`, `mc_efficient_frontier_years.png` | Монте-Карло и сравнение по годам |

Точный список зависит от выполненных ячеек.

## Модуль `risk_utils.py`

Содержит переиспользуемые функции: оценка μ/Σ, скользящие окна, `efficient_frontier_unconstrained`, границы max-дисперсии (long-only и с **gross leverage**), прокси ADV и **Almgren–Chriss**-штраф для задания 25, загрузка свечей MOEX, индексные ряды и др. Импорт в ноутбуке: `import risk_utils`.
